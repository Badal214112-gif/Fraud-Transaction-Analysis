# UPI Fraud Detection & Transaction Analytics Case Study

## Project Overview

This project is an end-to-end SQL and analytics case study focused on UPI transaction behavior and fraud detection. The objective of this project is to analyze transaction patterns, identify suspicious activities, generate business insights, and build an interactive dashboard for monitoring key metrics.

The complete workflow was implemented using Google BigQuery for data analysis and Looker Studio for dashboard development.

## Dataset Information

This project uses four interconnected datasets related to the UPI ecosystem.

**Transactions Dataset**
Acts as the primary dataset and stores transactional activities performed by users. It is used for transaction analysis, KPI generation, and fraud pattern identification.

**Users Dataset**
Contains customer-related information and helps analyze user behavior, segmentation, and risk patterns.

**Merchants Dataset**
Contains merchant-related information used to evaluate merchant activity and transaction behavior.

**Fraud Dataset**
Contains fraud indicators and risk labels used for fraud analysis and suspicious activity detection.

## Project Workflow

The project follows a structured analytics workflow:

1. Performed data validation to inspect data quality and identify inconsistencies.
2. Conducted data cleaning and preprocessing.
3. Generated business KPIs and transaction insights.
4. Performed user and merchant behavior analysis.
5. Identified fraud patterns and suspicious activities.
6. Applied advanced SQL techniques for deeper analysis.
7. Prepared dashboard-ready datasets.
8. Built an interactive dashboard in Looker Studio.
